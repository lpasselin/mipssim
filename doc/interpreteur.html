Module d’interprétation¶

Utilisation du module¶

Le module d’interprétation s’instancie avec le chemin du code source assembleur MIPS 64 bits en paramètre comme suit:

from src.interpreter import *
interpreter = mips_interpreteur(source_file)

L’interpréteur supporte l’étiquettage (label) d’une ligne.

La syntaxe supportée est la suivante:

[label:] Mnémonique  param1,param2[,param3]

Les éléments entre crochets sont optionnels.

Les paramètres supportés sont les suivants:

  • Registre entier (R0* à R31)
  • Registre flottant (F0 à R31)
  • Adresse mémoire (noté: offset(registre contenant l’adresse), ie: 0(R1) )
  • Étiquette (label) dans le cas des branch

* Attention au registre R0, c’est un registre à lecture seule.

Mnémoniques assembleur supportés¶

Les mnémoniques assembleur MIPS 64 bits supportés par l’interpréteurs sont les suivants:

  • LD
  • L.D
  • SD
  • S.D
  • ADD.D
  • SUB.D
  • MUL.D
  • DIV.D
  • DADD
  • DADDU
  • DADDI
  • DADDIU
  • DSUB
  • DSUBU
  • DMUL
  • DMULU
  • DDIV
  • DDIVU
  • AND
  • BEQZ
  • BNEZ
  • BEQ
  • BNE
  • J

Classes et membres du module d’interprétation¶

class src.interpreter.mips_interpreteur(source_file=None)[source]¶

Interpréteur de source assembleur du MIPS.

Entrée: Source en assembleur MIPS 64 bits. Sortie: Une liste. Chaque élément de la liste est une ligne de code.

Ces éléments sont des tuples de la forme suivante::
([“Unite_fonctionnelle”, “Operation_a_effectuer”], [“Param1”, “Param2”, ...])
parse_assembler(in_source)[source]¶

Convertis le code assembleur en mnémoniques représentés par l’énumération au début du fichier.

parse_instructions(in_source)[source]¶

Convertis une source composée des lignes de codes, labels exclus, en une liste de tuples représentant l’instruction puis ses paramètres opaques. ie. [(instruction_number, parameters), ...]

parse_labels(in_source)[source]¶

Cherche les labels dans le code et les assigne au dictionnaire labels de la classe sous la forme : labels[‘nom_du_label’] = numero_de_l’operation ou {‘nom_du_label’: numero_de_l’operation, ‘nom_du_label_2’: numero_de_l’operation_2}

parse_parameters(in_source)[source]¶

Segmente les paramètres des opérations en une liste d’éléments au sein de la liste de tuples retourné par la fonction parse_instructions.
